1. Método de la Secante

Conoce su origen Joseph Raphson & otros

2. Fundamento teórico ampliado

La secante aproxima la raíz de f(x) usando dos puntos anteriores xn-1 y xn para construir la línea secante y estimar su intersección con el eje x. No requiere derivada explícita.

Fórmula iterativa: xn+1 = xn - f(xn)·(xn-xn-1)/(f(xn)-f(xn-1)).

3. Término de error y convergencia

La convergencia es superlineal (orden ≈1.618), más rápida que bisección pero más lenta que Newton. El error cumple:

|xn+1-x*| ≈ C · |xn-x*|·|xn-1-x*|, (C depende de f'' y f')

4. Pasos detallados del algoritmo

  1. Elegir estimaciones iniciales x0, x1 cercanas a la raíz.
  2. Calcular xn+1 = xn - f(xn)*(xn-xn-1)/(f(xn)-f(xn-1)).
  3. Verificar tolerancia: |xn+1-xn|< tol o |f(xn+1)|< tol.
  4. Actualizar: xn-1=xn, xn=xn+1.
  5. Repetir hasta condición de parada.

5. Ejemplo resuelto

Aplicar secante a f(x)=x3-x-2 con x0=1, x1=2, tol=0.001:

  • Iteración 1: x2=2 - (4)/(4-(-2))=1.6667
  • Iteración 2: x3=1.6667 - (0.9629)/(0.9629-(-0.333))=1.5217
  • Iteración 3: x4=1.5217 - (0.219)/(0.219-(-0.126))=1.4969
  • Iteración 4: x5=1.4969 - (0.059)/(0.059-(-0.016))=1.4936
  • … raíz aproximada: x≈1.4934

6. Código en Python y gráfica

import numpy as np
import matplotlib.pyplot as plt

def secante(f, x0, x1, tol=1e-5, max_iter=50):
    xs = [x0, x1]
    for i in range(2, max_iter+2):
        x_prev, x_curr = xs[-2], xs[-1]
        f_prev, f_curr = f(x_prev), f(x_curr)
        x_next = x_curr - f_curr*(x_curr - x_prev)/(f_curr - f_prev)
        xs.append(x_next)
        if abs(x_next - x_curr) < tol:
            break
    return xs

# Ejemplo f(x)=x^3-x-2
def f(x): return x**3 - x - 2
its = secante(f, 1, 2, tol=1e-3)
errs = [abs(x - its[-1]) for x in its]

plt.figure()
plt.plot(range(len(errs)), errs)
plt.xlabel('Iteración')
plt.ylabel('Error estimado')
plt.title('Convergencia del Método de la Secante')
plt.show()

10. Conclusión y aplicaciones

El método de la secante ofrece convergencia superlineal sin requerir derivada, combinando velocidad y simplicidad. Sin embargo, su estabilidad depende de buenas estimaciones iniciales.

Se emplea cuando la derivada es difícil de calcular o costosa, en física e ingeniería para resolver ecuaciones no lineales.
